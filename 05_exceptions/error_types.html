Types of Errors in Python

Syntax Errors

Syntax errors, also known as parsing errors, is the most common type of errors you may have encountered so far while using Python. For example:

If I try to print the text, Hello World, without using quotation marks, then we get a SyntaxError. Anything within the brackets (or argument), that does not have quotation marks, Python expects it to be a variable with no spaces. If I try to print the text, Hello, without using quotation marks, Python doesn’t realize it is a syntax error but instead thinks we are trying to print a variable called Hello that we haven’t defined.

Simply put, syntax errors arise when Python is unable to understand a line of code. Usually, it caused by some missing character or space in order for the piece of code to execute successfully.

Logic Errors

Logic Errors are the most difficult type of error to fix in Python. They occur when the program runs without crashing, but it produces an incorrect result. Common sources of logic errors include:

  • Misspelled identifiers
  • Variable mix-ups
  • Incorrect indentation (usually caused by copying and pasting large blocks of code)

Sometimes PEP8 format checkers will catch logic errors, but often you have to debug the program by trial and error to find the problem. We will learn more about logic errors later in the semester.

Exceptions

You may already be familiar with the following exception errors: ValueError, ZeroDivisionError, and NameError.

ValueError - an operation or function receives an argument with an inappropriate value. For example:

# input an integer
x = int(input("Please input a number: "))

We’re expecting the user to input an integer. If we don’t “handle the exception”, the program crashes if the user inputs a float or string characters.

ZeroDivisionError - the second argument of a division or modulo is zero. Unless we tell our program how to “handle” division by zero, the program crashes.

NameError - a local or global variable is not found. If we forget to define a variable, when we try to use it, the program crashes. Like the SyntaxError above, sometimes if we don’t put quotation marks around our text, then Python thinks it is a NameError. Python has many other built-in exceptions: https://docs.python.org/3/library/exceptions.html. You can skim through most of the list but make sure to read the section on OS exceptions. We will be dealing with OS exceptions when we are file handling in Python.
