Objects and Classes

Practice:

  • Create a class name Triangle
  • Create an object from it. the object will have three attributes names a, b, c that represent the sides of the triangle.
  • The Triangle class will have two methods:
    • the init method to initialize the sides
    • a method to calculate the perimeter of the triangle from its sides
  • The perimeter of the triangle should be printed from outside the class

Let's consider a class called Hero for an RPG:

Here is a program that calculates the the cost of a movie, drink, and popcorn. You will notice that there are attributes like __dollars and __cents where the __ tells the program that these are private and cannot be accessed outside the class definition. We would make attributes private to protect them from being changed from their initial value.

You can see that in line 39,

drink_price.youwish = drink_price.__dollars * 100000

__dollars is outside the class definition. If you run the program you will receive

Traceback (most recent call last):
File "main.py", line 39, in <module>
drink_price.youwish = drink_price.__dollars * 100000
AttributeError: 'Currency' object has no attribute '__dollars'

because the program does not recognize __dollars outside the class.
